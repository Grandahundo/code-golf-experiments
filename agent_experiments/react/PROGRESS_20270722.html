发现一：难度分类确实能区分 AI 和人类

chenhao 的测试集里面，难度确实很能区分 deepseek-v4-pro 和人类。AI/Human 比值随难度递增——简单题仅 1.18 倍，困难题达到 2.81 倍。

IRT 评级说明

IRT 算出来的评级也很能说明问题。那么 IRT 评级到底是什么？

IRT（项目反应理论）评级 — 来源与含义

IRT 评级来自 difficulty/grm_item_params.csv，基于 GRM (Graded Response Model，等级反应模型) 对 48 支参赛队伍在所有 400 个任务上的压缩表现拟合得出。每个任务有 5 个参数：

参数含义与 AI 的相关性
a (区分度)题目区分高能力和低能力受试者的精度。高 a = 题目能精准区分能力强弱ρ = −0.04 — 几乎无关
b₁–b₄ (难度阈值)将连续能力 θ 映射到 5 个有序响应类别的 4 个分界点。b₁ 是最低门槛，b₄ 是最高b₂ (ρ=0.76)、b₃ (ρ=0.63) 相关性较高
b₄−b₁ (难度跨度)题目覆盖的能力范围ρ = −0.02 — 几乎无关
info_mean / info_high平均信息量 / 高能力区间信息量info_high (ρ=0.61) 有一定相关性

IRT 等级 (A/B/C)

等级基于 a 区分度 划分：A 级 (a 高，区分度极好)、B 级 (良好)、C 级 (可接受)。 IRT 难度层级 (T1_easiest ~ T5_hardest) 则基于 b 参数综合排序。

难度分类 × IRT 等级交叉表

难度分类A (高区分)B (良好)C (可接受)合计
简单27716
中等3612048
困难1517032

关键观察：简单题散布在 A/B/C 各等级，中等问题几乎全是 A 级（高区分度），困难题以 A/B 为主。 这意味着 selection 的"简单/中等/困难"分类和 IRT 区分度不是同一个维度—— 简单题虽然容易，但有些也是高质量区分器（A 级），有些区分度低（C 级，大家都做得差不多）。

发现二：IRT 等级高 → 模型间差异大

IRT 评级高的题目在 qwen-3.6-flash 和 deepseek-v4-pro 上有比较大的差异。 IRT Grade A 的题，QW/DS 比值达到 1.88x；Grade C 的题只有 1.16x。

发现三：ReAct + DS V4 Pro 整体不如 Claude Code

我的 ReAct 框架搭配我的提示词用 deepseek-v4-pro 跑出来的效果整体是不如 claude code 和 codex 的。

我的 ReAct 框架比较简单，不支持文件读写，所以我自己弄了一套 prompt，和 chenhao 的没有对接上，因此我的实验跟他之间还不好直接对比。
