# Code-golf API 实验数据总结

*2026-07-17 · 模型：deepseek-v4-pro / glm-5.2 / qwen3.7-max（均开启思考模式）*

**方法**：每次 trial 向 API 发送同一模板 prompt（任务说明 + 该题 train 样例网格），从回复中提取 ```python 代码块，对 train+test 全部样例做一次正确性测试，统计字节数。长度统计仅计正确解。

## 实验一：golf prompt（要求"最短"），task001 × 20 trials/模型

| 模型 | 正确率 | 长度 min / mean ± std (B) | 耗时中位 |
|---|---|---|---|
| deepseek-v4-pro | 17/20 | 66 / 77.8 ± 7.6 | 272s |
| glm-5.2 | 18/20 | 65 / 89.8 ± 15.0 | 35s |
| qwen3.7-max | **20/20** | 65 / **73.1** ± 8.8 | 419s |

三家最短解几乎打平（65–66B）；qwen 全对但最慢，glm 最快但长度方差最大。deepseek 的 3 次错误完全相同（92B、0/6），是同一错误规则的吸引子。

## 实验二：no-golf prompt（不要求最短），task001–010 × 5 trials/模型

| 模型 | 正确率 | 长度中位 (B) |
|---|---|---|
| deepseek-v4-pro | 42/50 (84%) | 1253 |
| glm-5.2 | 38/50 (76%) | 742 |
| qwen3.7-max | **49/50 (98%)** | 944 |

按题（3 模型合计 15 trials）：正确率 task005 8/15 < task004 9/15 < task007 11/15 < task003 = task009 13/15 < 其余五题 15/15；正确解中位长度从 task001 的 425B 到 task005 的 3190B，跨约 **7.5×**。

## 结论

1. **"写最短"一句话值 ~5 倍长度**：task001 上 golf 解 65–78B vs 自然解中位 425B。
2. **自然长度是可用的难度信号**：最难三题（005/004/007）恰为最长的题；错误率与长度排序基本一致。
3. **同题稳定性尚可但随难度劣化**：多数 题×模型 的长度 CV 在 9–25%，难题上个别达 ~70%。
4. **qwen3.7-max 两种 prompt 下都最稳**（20/20 与 49/50，唯一失败是 1 次 API 错误），代价是耗时最长；glm 快而不稳；deepseek 居中且错误模式高度一致。
